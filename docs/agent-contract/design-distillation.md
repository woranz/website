# Design Distillation

Use this file to remove explanatory excess, filler copy, and decorative structure that does not earn its place.

## Canonical Rule

If a sentence, block, or container can be removed without increasing confusion, remove it.

The purpose of `distill` is not to make the UI colder. It is to make meaning land faster.

## Distill Checks

### Explain Only The Non-Obvious

Keep text when it:

- prevents a likely mistake in a quoter or form
- clarifies a consequence the user could not infer (coverage limits, exclusions)
- reduces anxiety around a real decision (price, commitment)
- introduces a useful shortcut or default

Remove text when it:

- repeats the headline in other words
- repeats the CTA in other words
- narrates what the page already shows
- adds "helpful" insurance explanation without changing a decision

### Hero Distillation

Default hero should fit this structure:

1. headline (Noe Display, one strong value proposition)
2. one short supporting line
3. one primary CTA

Optional extras must earn their place:

- one brief trust signal
- one secondary action
- one contextual number or stat

Do not add a fourth block by reflex.

### Section Distillation

Each page section should earn its scroll:

- Does this section answer a question the user actually has at this point?
- Would removing it make the conversion path unclear?
- Is it duplicating information from another section?

### Container Distillation

Remove or collapse containers when they:

- group no real decision
- restate hierarchy that spacing already communicates
- exist only to feel "finished"

## Output Contract

UI work must produce `Distill Notes` that answer:

- What copy or chrome got removed?
- Which line or block survived because it reduced real uncertainty?
- Is the hero still clear after the cuts?

## Blocking Smells

Reject the proposal if:

- the page explains what the CTA already says
- helper text is longer than the decision it supports
- a section adds a second explanatory block without a clear uncertainty it resolves
- the page feels "professional" only because it is over-explaining
