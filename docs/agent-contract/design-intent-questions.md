# Visual Intent Questions

Use this file when a UI brief says the work should feel:

- less corporate or more human
- more Woranz
- less generic
- more trustworthy
- more premium
- warmer
- calmer
- more direct

without explaining what should become more obvious in the page itself.

## Canonical Rule

When the brief contains vague quality adjectives but weak visual direction, do not jump straight into layout.

Pause and surface 2 or 3 `Blocking Visual-Intent Questions` before `critique`.

The agent should answer them from repo context when possible. Ask the user only if the answer is materially risky and the repo cannot resolve it.

## When This Is Mandatory

Run this step when any of these are true:

- the request asks for a page to feel less corporate, less flat, less generic, or more Woranz
- the request asks for "better design" without naming the intended dominant read
- the request says the page should be more trustworthy or warmer but does not say where that quality should come from
- the task is a redesign or visual fix on an existing page whose business goal is still underspecified

## Question Categories

Choose 2 or 3 questions that will actually change the direction.

Good categories:

- dominant read
  - what must become obvious in 3 seconds?
- page purpose
  - is this page primarily conversion-focused, informational, exploratory, or comparison?
- warmth source
  - should the warmth come from copy, imagery, team presence, specific numbers, or product detail?
- conversion hierarchy
  - what action or product should feel primary?
- segment angle
  - is the framing for personas (personal), empresas (business), or productores (agent)?
- density tradeoff
  - should this page feel more immediate, more spacious, or more content-rich?

## Bad Questions

Do not ask questions that only restate the brief.

Avoid:

- "Do you want it better?"
- "Should it look warmer?"
- "Do you want more Woranz vibe?"

## Output Contract

When this step is mandatory, UI-stage work must include:

- `Blocking Visual-Intent Questions`

That section should show:

- the 2 or 3 questions
- the resolved answer for each
- whether the answer came from repo evidence or a user clarification
